# MarioClock 软件架构文档

本文档描述了 MarioClock 项目的软件架构、模块设计和核心逻辑实现。

## 1. 项目概述
MarioClock 是一个基于马里奥主题的数字时钟仿真器。它模拟了马里奥在时钟界面中活动，并在时间改变时通过碎裂特效更新数字。项目采用 C 语言编写，支持跨平台硬件抽象层（HAL），当前实现了 PC 平台的 SDL2 仿真。

## 2. 软件架构层级
项目采用分层架构设计，确保了逻辑与硬件的解耦。

### 2.1 硬件抽象层 (HAL - Hardware Abstraction Layer)
*   **文件**: [`hal.h`](include/hal.h), [`hal_pc.c`](src/hal_pc.c)
*   **职责**:
    *   管理图形显示初始化与帧缓冲更新（基于 SDL2）。
    *   获取用户输入（键盘映射为游戏按键）。
    *   提供系统滴答计时（Ticks）和延时（Delay）功能。
*   **接口**: `HAL_Init()`, `HAL_UpdateScreen()`, `HAL_GetInput()`, `HAL_GetTicks()`, `HAL_Delay()`。

### 2.2 渲染引擎层 (Engine Layer)
*   **文件**: [`engine.h`](include/engine.h), [`engine.c`](src/engine.c)
*   **职责**:
    *   管理屏幕缓冲区（Screen Buffer）。
    *   提供基础的精灵（Sprite）绘制功能，支持水平镜像翻转。
    *   支持从图集（Atlas）中切片绘制，用于数字和碎屑的渲染。
*   **关键数据结构**: `Animation`, `Character`（虽然包含物理属性，但渲染依赖其状态）。

### 2.3 功能模块层 (Functional Modules)
项目重构后采用模块化设计，每个模块负责独立职责：

#### 2.3.1 角色模块 (Character Module)
*   **文件**: [`mario.h`](include/mario.h), [`mario.c`](src/mario.c)
*   **职责**: 
    *   管理马里奥角色的物理状态（位置、速度、加速度）
    *   处理用户输入并更新角色运动
    *   管理角色动画状态和帧切换
*   **关键函数**: `Mario_Init()`, `Mario_UpdateInput()`, `Mario_UpdatePhysics()`, `Mario_UpdateAnimation()`, `Mario_Render()`

#### 2.3.2 粒子系统 (Particle System)
*   **文件**: [`particle.h`](include/particle.h), [`particle.c`](src/particle.c)
*   **职责**:
    *   管理粒子对象池（最多 150 个粒子）
    *   处理数字碎裂特效的粒子生成和物理更新
    *   粒子生命周期管理（屏幕优化剔除）
*   **关键函数**: `Particle_Init()`, `Particle_SpawnBrickDebris()`, `Particle_ShatterDigit()`, `Particle_Update()`, `Particle_Render()`

#### 2.3.3 数字显示模块 (Digit Display Module)
*   **文件**: [`digit.h`](include/digit.h), [`digit.c`](src/digit.c)
*   **职责**:
    *   管理 4 个时间数字的显示状态机（常亮、碎裂、闪烁）
    *   检测时间变化并触发对应特效
    *   控制冒号闪烁效果
*   **关键函数**: `Digit_Init()`, `Digit_InitFromTime()`, `Digit_Update()`, `Digit_Render()`

#### 2.3.4 游戏逻辑层 (Game Logic Layer)
*   **文件**: [`game.h`](include/game.h), [`game.c`](src/game.c)
*   **职责**:
    *   集中管理游戏的全局状态
    *   实现任务调度的具体逻辑
    *   协调所有功能模块的交互
*   **关键函数**: `Game_Init()`, `Game_StartFrame()`, `Game_TaskInput()`, `Game_TaskPhysicsUpdate()`, `Game_TaskUpdateStateMachine()`, `Game_TaskRender()`, `Game_TaskFrameRateControl()`

### 2.4 主程序入口层 (Main Entry)
*   **文件**: [`main.cpp`](src/main.cpp)
*   **职责**:
    *   初始化游戏
    *   实现任务调度循环（60FPS 轮询）
    *   按优先级顺序执行各任务

### 2.5 数据资源层 (Data/Resources)
*   **文件**: [`mario_sprites.c`](src/mario_sprites.c), [`include/mario_sprites.h`](include/mario_sprites.h), [`include/sprite.h`](include/sprite.h)
*   **内容**: 包含马里奥及其它精灵图的 RGB565 像素数据数组。

## 3. 核心机制设计

### 3.1 任务调度框架
主循环（`main.c`）采用**轮询式任务调度**设计，每帧按优先级执行以下任务：

```
┌─────────────────────────────────────────────┐
│  Frame Start (记录 tick)                     │
└─────────────────────────────────────────────┘
           ↓
┌─────────────────────────────────────────────┐
│  Task 1: Input (输入处理)                    │
│  - 读取按键、更新角色速度和方向              │
└─────────────────────────────────────────────┘
           ↓
┌─────────────────────────────────────────────┐
│  Task 2: Physics Update (物理更新)           │
│  - 更新位置、速度、碰撞检测                  │
│  - 更新动画帧索引                            │
└─────────────────────────────────────────────┘
           ↓
┌─────────────────────────────────────────────┐
│  Task 3: State Machine Update (状态机更新)   │
│  - 检测时间变化、触发碎裂特效                │
│  - 更新数字状态机（常亮→碎裂→闪烁→常亮）    │
│  - 更新粒子物理                              │
└─────────────────────────────────────────────┘
           ↓
┌─────────────────────────────────────────────┐
│  Task 4: Render (渲染)                       │
│  - 清空缓冲区                                │
│  - 分层绘制（冒号 → 数字 → 粒子 → 角色）    │
│  - 上传到屏幕驱动                            │
└─────────────────────────────────────────────┘
           ↓
┌─────────────────────────────────────────────┐
│  Task 5: Frame Rate Control (帧率控制)       │
│  - 锁定 60 FPS (16 ms/frame)                │
└─────────────────────────────────────────────┘
```

**优势**：
- 代码逻辑清晰，易于维护和扩展
- 优先级顺序明确，便于调试
- 可灵活添加新的任务类型

### 3.2 粒子碎裂系统
当分钟或小时发生变化时，系统会计算发生变化的数字位。
1.  **触发**: `Particle_ShatterDigit()` 将 3x5 的数字阵列拆解为独立方块。
2.  **生成**: `Particle_SpawnBrickDebris()` 为每个方块生成 4 个小的 8x8 碎屑粒子。
3.  **运动**: 粒子被赋予初始爆炸速度和重力加速度。
4.  **回收**: 粒子超出屏幕底部后自动释放。

### 3.3 数字状态机
每个数字位由一个 `TimeDigit` 结构体维护其状态：
*   `STATE_IDLE (0)`: 静态显示数字砖块。
*   `STATE_SHATTER (1)`: 触发碎裂特效后的空白期（1000ms 粒子飞散）。
*   `STATE_FLASH (2)`: 数字重新生成前的闪烁期（600ms，每 80ms 切换可见性）。

**状态转移**：
```
常亮 (0) ──时间变化→ 碎裂 (1) ──1000ms→ 闪烁 (2) ──600ms→ 常亮 (0)
```

### 3.4 物理与动画
*   **马里奥物理**: 采用简化的欧拉积分处理速度和位置，物理常量定义在 `mario.c` 中。
*   **动画切换**: 根据马里奥的状态（在空中、移动、静止）动态切换动画序列帧。
*   **平滑度**: 使用 `distance_moved` 计步器确保马里奥走路动画与移动速度成正比。

## 4. 模块依赖关系
```
main.c (任务调度器，最少的职责)
   ↓
game.c (集中的游戏逻辑)
   ↓
┌─────────────────────────────────────────┐
│  mario.c      particle.c      digit.c   │
│  (角色模块)   (粒子模块)    (数字模块)   │
└─────────────────────────────────────────┘
   ↓
┌─────────────────────────────────────────┐
│  engine.c (渲染引擎)                     │
└─────────────────────────────────────────┘
   ↓
┌─────────────────────────────────────────┐
│  hal.c (硬件抽象层)                      │
└─────────────────────────────────────────┘
```

**设计原则**：
- 底层模块不依赖上层模块
- 功能模块相对独立，通过 `game.c` 进行协调
- 清晰的接口定义（头文件），隐藏实现细节

## 5. 编译与环境
*   **构建工具**: PlatformIO
*   **平台**: `native` (PC Simulator)
*   **依赖**: SDL2 (开发库需安装在系统中)
*   **编译宏**: `PC_SIMULATOR` 用于启用 PC 版本的 HAL 实现。

## 7. 开发建议

### 7.1 添加新功能
如需添加新的游戏机制（如敌人、道具、音效等），建议：
1. 为新功能创建独立的模块文件（如 `enemy.h/c`）
2. 在 `game.c` 中添加相应的初始化和任务函数
3. 在 `main.cpp` 的调度循环中插入新任务

### 7.2 修改任务优先级
编辑 `main.cpp` 中 while(1) 循环内的任务顺序即可，无需修改其他文件。

### 7.3 调试与性能分析
- 利用 `HAL_GetTicks()` 进行帧时间测量
- 在各任务间插入计时代码以识别性能瓶颈
- 利用模块化设计，抽离相关代码到单独文件进行单元测试
